    SharedPreferences 源码解析

    本文源码分析基于 API26

    一、概述

    Android 中存储数据一共有三种方式：存储键值对集合，通过文件存储以及通过SQL数据库存储。

    其中，保存较小键值对集合就采用 SharedPreferences 实现。SharedPreferences 本质就是一个 xml文件。

    SharedPreferences 是 访问 和 修改 包含 键值对 的文件的接口。每个 SharedPreferences 文件由文件系统框架进行管理并且可以被共享。

    1. 初始化。根据指定文件名初始化 SharedPreferences 时候，需要指定另一个参数 mode，它有四个可选配置：
      • MODE_PRIVATE : 当前文件只能被本应用或者其他和当前应用共享 user ID的应用访问和修改
      • MODE_WORLD_READABLE : 当前文件对所有应用都开放读权限。已经被弃用，推荐使用 FileProvider，Android N 版本使用会抛出 SecurityException
      • MODE_WORLD_WRITEABLE : 当前文件对所有应用都开放读写权限。已经被弃用，推荐使用 FileProvider，Android N 版本使用会抛出 SecurityException
      • MODE_MULTI_PROCESS : 同应用多进程安全访问。已经被启用，在一些版本上并不可靠并且不再维护，对于多进程共享数据推荐使用 ContentProvider

    3. 写入。在写入数据后记得提交本次写入内容，方法为 apply() 或 commit() (两个方法的区别会在后文中讲到)

    分析：

    • 对于每个 SharedPreferences 操作，所有数据都是存储在内存中
    • 对于 putString(String key, @Nullable String value) 和 putStringSet(String key, @Nullable Set<String> values) 两个方法，如果对应的 value 为 null 的话，相当于移除掉对应的 key 的配置(等价于 remove(String key) )
    • 对于每次修改，需要调用 apply() 或者 commit() 才能将提交的修改写入内存和磁盘中
      • apply() : 异步写入。效率高。
      • commit() : 同步写入。返回写入结果。多次写入会阻塞等上次写入磁盘完成，效率低。
    • 每次写入都会将当前 SharedPreferences 所有数据全部写入 xml文件

    三、小结

    对于简单数据使用，直接使用 SharedPreferences 会比较方便，其本质就是在应用文件夹下一个存储数据的 xml文件 。
